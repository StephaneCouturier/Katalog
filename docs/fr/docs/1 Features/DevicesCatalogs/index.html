Devices: Catalogs

Summary​

  • This page describes all the features of the Catalogs screen and how to use them.
  • The Catalogs screen lists all catalogs of the current Collection folder
  • Each catalog is an index of files from a given location called source path.
  • The row of buttons is enabled once a catalog is selected, allowing various actions on the selected catalog.

List and selection​

The list of catalogs can be limited by using the Filters Location and Storage in the left panel. Those values are set in the screen Storage and by associating a catalog to a Storage Device.

Actions buttons​

The buttons (act on selected catalog)

  • Search: Select the catalog as a filter (left panel, Filters tab), and goes to the Search screen, to quickly search files in that specific catalog.
  • Explore: Open the selected catalog file in the Explore screen to view the catalog's folders and files.
  • View: View the statistics of the selected catalog in the Statistics screen.
  • Edit: Opens a panel to change name, path, storage device, etc. See details below before modifying any field to understand the consequences.
  • Update: Update the selected catalog by listing all files again from its source path, according to its criteria (type of file, hidden files, etc.)
  • All active: Update all the displayed catalogs that are active (its device is connected)
  • Delete: Delete the catalog. (This does not delete the backup file, nor the related values in the statistics.csv)
  • Snapshot: Record all catalogs values (name, size, files, etc.) indepentanly of the filters. These records support the creation of [Statistics](https://github.com/StephaneCouturier/Katalog/wiki/Statistics
  • Import: Imports a catalog from another tool. Currently import from VVV is supported.

Catalog List​

  • If the source path of a catalog points to storage device that is active/connected/mounted, the icon and catalog name is colored in green.

Import​

It is now possible to import all VVV physical volumes from one export file made using tabulation as separator.
Each VVV physical volume will become 1 separate catalog.
Limit: the source path and other info about the volume are not available in the exportfrom VVV.

  • In VVV, choose File / Export..., and select TAB as Separator
  • From Katalog, screen Catalogs: click the Import button and select the file created previously.
  • Using the Edit button, it is possible to add the catalog source path to enable updates later.

Edit​

It is generally recommended to Create a new catalog with the right initial settings: source path, file type, include Hidden Files, and storage, rather than modify the catalog file directly. Catalog information (source path, file type, etc.) can be modified.

Rename: Renames the selected catalog file. The catalog can be renamed in the statistics file.

Technical information about catalog definition:​

The definition of the catalog and the list of files itself are all stored into the catalog file. Each catalog file contains first a series of mandatory lines:

<catalogSourcePath> + path of the source folder to be cataloged

<catalogFileCount> + number of files (it is calculated and update by Katalog when updating the catalog)

<catalogTotalFileSize> + total size of the files (it is calculated and update by Katalog when updating the catalog)

<catalogIncludeHidden> + true or false

<catalogFileType> + one of the values: All, Audio, Image, Text, Video

<catalogStorage> + name_of_the_storage

This is then followed by the list of files, with 1 file information per line, file field are tab separated.

Changing catalogSourcePath, catalogIncludeHidden, catalogFileType, catalogIncludeSymblinks impact and define which files will be listed in the catalog. The list of files for the catalog will matching any change after running the Update on this catalog.
